Brief Introduction

Order from Chaos

An algorithm is simply a step-by-step instruction set to solve a problem.

Without them, data would be a messy pile. With them, we transform raw data into actionable information.

Two Pillars

  • Sorting: Arranging data (A-Z, 1-100).
  • Searching: Finding specific data efficiently.

If you can't sort it, you can't search it efficiently.

Why Are They Needed?

1. Why Sort?

[9, 2, 5, 1] ➝ [1, 2, 5, 9]
  • Performance: Pre-req for efficient searching.
  • Readability: Users need price (Low-High) or Name (A-Z).
  • Optimization: Identifying duplicates easily.

2. Why Search?

Find 'User 404' in 1 Billion rows
  • Data Retrieval: Database queries, logins, file systems.
  • Speed: Navigating without checking every single item.
  • Connectivity: Routing packets on the internet.

Sorting Algorithms Explained

🫧 Bubble Sort

Step through the list, compare neighbors, swap if wrong.

Simple but Slow O(n²)

⚡ Merge Sort

"Divide and Conquer". Split list in half recursively, then merge sorted halves.

[8 3 2 9]
⬇
[8 3] [2 9]
⬇
[3 8 2 9]
Consistent O(n log n)

🚀 Quick Sort

Pick a "pivot". Throw smaller items to left, larger to right. Repeat.

⬅ Smaller | Pivot | Larger ➡
Very Fast usually O(n log n)

How Searching Works

🐌 Linear Search

Check index 0. Is it X? No.
Check index 1. Is it X? No.
Check index 2...

for (i in list) if i == x return true

Inefficient: O(n)

🧠 Binary Search

Requires Sorted list.
Go to middle. Is X bigger? Ignore left half.

while low <= high: mid = (L+H)/2...

Efficient: O(log n)
